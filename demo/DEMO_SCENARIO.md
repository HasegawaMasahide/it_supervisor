# デモシナリオ: IT資産監査サービス

## シナリオ概要

**顧客プロファイル**: ACME株式会社
- 業種: Eコマース
- 従業員数: 50名
- 課題: 5年前に構築したWebアプリケーションの保守性とセキュリティに不安

**デモの目的**: IT資産監査サービスの価値を示し、契約につなげる

## Q&A 想定問答

### Q1: 「AIが実装したコードは本当に信頼できるのか？」

**A**:
「素晴らしい質問です。弊社では、AIが生成したコードに対して**4層の品質ゲート**を設けています：

1. **自動検証**: AI自身がコードをテストし、基本的な問題を修正
2. **エンジニアレビュー**: シニアエンジニアが全コードをレビュー
3. **統合テスト**: 実際の環境でテストを実行
4. **顧客検証**: 御社の環境で動作確認

さらに、Critical問題の修正については、必ず2名以上のエンジニアがレビューします。これにより、AIの効率性と人間の判断力を組み合わせた高品質なコードを提供できます。」

### Q2: 「瑕疵担保の範囲はどこまでか？」

**A**:
「瑕疵担保の対象は以下の3つです：

1. **コード変更による機能破壊**: 弊社の修正により既存機能が動かなくなった場合
2. **既存機能の退行**: 新しいコードが原因で、以前は動いていた機能に問題が発生した場合
3. **セキュリティ診断の重大な見落とし**: Critical問題を見逃していた場合

ただし、以下は対象外です：
- 御社による追加開発や設定変更
- インフラ障害や外部サービスの問題
- 仕様変更に伴う追加開発

期間は**実装完了後3ヶ月間**です。」

### Q3: 「社内の機密情報は安全に扱われるのか？」

**A**:
「セキュリティは最優先事項です。以下の対策を実施しています：

1. **NDA締結**: プロジェクト開始前に秘密保持契約を締結
2. **アクセス制限**: 必要最小限のメンバーのみがコードにアクセス
3. **暗号化**: 解析データは全て暗号化して保存
4. **監査ログ**: 全てのアクセスを記録し、定期的にレビュー
5. **プロジェクト終了後の削除**: 契約終了後、全てのデータを削除

また、必要に応じてオンプレミス環境での解析も対応可能です（追加費用が発生する場合があります）。」

### Q4: 「他社と比べて料金が高いのでは？」

**A**:
「一見高く見えるかもしれませんが、実は**トータルコストでは安価**です。

他社の場合：
- 診断会社に80万円（診断のみ）
- 開発会社に200万円（実装）
- **合計: 280万円**

弊社の場合：
- 診断+実装+効果測定: **120万円**
- **節約額: 160万円（57%削減）**

さらに、弊社は一気通貫で対応するため、診断結果の伝達ミスや、実装時の認識齟齬がありません。これにより、手戻りが大幅に削減されます。」

### Q5: 「プロジェクトの途中でキャンセルできるか？」

**A**:
「はい、可能です。ただし、フェーズごとの区切りでお願いしています：

- **フェーズ1-3（調査〜診断）完了後**: 実装を行わずに終了可能（料金の40%）
- **フェーズ4（提案）完了後**: 実装を行わずに終了可能（料金の60%）
- **フェーズ5（実装）開始後**: 実装済み部分に応じた料金をお支払いいただきます

弊社としては、診断結果を見てから実装を判断していただいても構いません。まずは診断だけでも価値があると自負しています。」
