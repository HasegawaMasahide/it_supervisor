## セキュリティ問題（17件）

### Critical（6件）

#### 1. APIキーのハードコーディング
- **ファイル**: `src/services/api.ts:4-5`
- **問題**: API KeyとAPI Secretが平文でソースコードに記載
- **影響**: 認証情報の漏洩、不正アクセス
- **修正工数**: 2時間
- **修正方法**: 環境変数（.env）に移行

#### 2. 認証トークンのlocalStorage保存
- **ファイル**: `src/services/api.ts:26-27`
- **問題**: 認証トークンとユーザー情報をlocalStorageに平文保存
- **影響**: XSS攻撃時にトークン窃取、セッションハイジャック
- **修正工数**: 4時間
- **修正方法**: HttpOnlyクッキーの使用、トークンの暗号化

#### 3. パスワードのログ出力
- **ファイル**: `src/services/api.ts:49`
- **問題**: ユーザー登録時にパスワードを含むデータをconsole.logで出力
- **影響**: パスワードの漏洩（ブラウザDevTools、ログファイル）
- **修正工数**: 1時間
- **修正方法**: 機密情報のログ出力を削除

#### 4. XSS脆弱性（dangerouslySetInnerHTML）
- **ファイル**: `src/components/ArticleEditor.tsx:155-157`
- **問題**: ユーザー入力をサニタイズせずにHTMLとして描画
- **影響**: スクリプトインジェクション、情報窃取
- **修正工数**: 3時間
- **修正方法**: DOMPurifyでサニタイズ、またはMarkdownパーサー使用

#### 5. XSS脆弱性（innerHTML直接操作）
- **ファイル**: `src/components/ArticleEditor.tsx:135-140`
- **問題**: DOM操作でinnerHTMLに直接ユーザー入力を代入
- **影響**: スクリプトインジェクション
- **修正工数**: 2時間
- **修正方法**: textContentの使用、またはReactの仮想DOMを使用

#### 6. eval()の使用
- **ファイル**: `src/utils/helpers.ts:2-4`
- **問題**: eval()で任意のコードを実行可能
- **影響**: リモートコード実行、システム侵害
- **修正工数**: 3時間
- **修正方法**: eval()の完全削除、安全な代替手段の検討
