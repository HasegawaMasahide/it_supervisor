# IT資産監査サービス - デモパッケージ

このディレクトリには、IT資産監査・改善サービスのデモンストレーション用の資産が含まれています。

## 📁 構成

```
demo/
├── setup.sh                    # デモリポジトリ復元スクリプト（clone後に実行）
├── bundles/                    # 各デモプロジェクトのGit履歴（bundleファイル）
├── project-docs/               # 各デモプロジェクトのREADME.md/ISSUES.md
│
├── README.md                    # このファイル
├── ANALYSIS_GUIDE.md           # 解析実行手順書
├── DEMO_SCENARIO.md            # プレゼンテーションシナリオ
├── REPORT_TEMPLATE.md          # レポートテンプレート
│
├── laravel-todo-app/           # 解析対象アプリ1: PHP/Laravel TODO（setup.sh実行後に生成）
├── react-cms-app/              # 解析対象アプリ2: React CMS（やや古い）
├── django-ec-app/              # 解析対象アプリ3: Django EC（やや古い）
├── aspnet-legacy-system/       # 解析対象アプリ4: ASP.NET Core レガシー社内システム
├── vue-booking-app/            # 解析対象アプリ5: Vue.js 予約管理システム
├── springboot-crm-app/         # 解析対象アプリ6: Spring Boot 顧客管理システム（CRM）
│
├── scripts/                    # 解析スクリプト
│   └── analyze-todo-app.ts    # メイン解析スクリプト
│
├── data/                       # 解析結果データ（実行後に生成）
└── reports/                    # 生成されたレポート（実行後に生成）
```

## 🎯 デモの目的

このデモパッケージは、以下の目的で使用できます：

1. **顧客へのサービス説明**: 実際のツールを動かして、具体的な価値を示す
2. **社内トレーニング**: 営業担当者やエンジニアがサービスを理解する
3. **技術検証**: ツール群の動作確認と改善点の洗い出し
4. **提案書作成**: デモ結果をもとに、具体的な提案書を作成

## 🚀 クイックスタート

### 1. デモリポジトリの復元

デモプロジェクトのソースコードはgit bundleとして管理されています。
リポジトリをcloneした後、以下を一度だけ実行してください:

```bash
bash demo/setup.sh
```

これにより、6つのデモプロジェクトがGit開発履歴付きで復元されます。
各プロジェクトのREADME.mdとISSUES.mdも自動的に配置されます。

### 2. 環境準備

```bash
# 必要なツールのインストール（詳細はANALYSIS_GUIDE.mdを参照）
# - Node.js v18+
# - TypeScript
# - PHP_CodeSniffer
# - Snyk
# - Gitleaks
```

### 3. 解析対象アプリの確認

```bash
cd laravel-todo-app
git log --oneline   # 開発履歴を確認
cat README.md       # 含まれる問題の概要を確認
```

### 4. 解析の実行

```bash
cd scripts
npx ts-node analyze-todo-app.ts
```

### 4. レポートの確認

```bash
# HTMLレポートを開く（ブラウザで開きます）
start ../reports/analysis-report.html  # Windows
open ../reports/analysis-report.html   # macOS
xdg-open ../reports/analysis-report.html # Linux
```

## 🔧 カスタマイズ

## 💡 プレゼンのポイント

### 強調すべき3つの価値

1. **スピード**: AI活用で従来の半分の期間で完了
   - 例: 「この解析、手動なら2週間かかりますが、ツールなら10分です」

2. **品質**: 人間によるレビューで品質保証
   - 例: 「AIが検出した問題を、シニアエンジニアが全てレビューします」

3. **安心**: 瑕疵担保責任で不安を解消
   - 例: 「もし弊社の修正に問題があれば、3ヶ月間無償対応します」

### よくある質問と回答

**Q: AIが実装したコードは信頼できるのか？**
A: 4層の品質ゲート（自動検証→エンジニアレビュー→統合テスト→顧客検証）で品質を保証します。

**Q: 機密情報は安全か？**
A: NDA締結、アクセス制限、暗号化、監査ログで対応。プロジェクト終了後はデータを削除します。

**Q: 他社より料金が高いのでは？**
A: 診断と実装を別々に発注すると280万円。弊社は一気通貫で120万円なので、実は57%安いです。

詳細は [DEMO_SCENARIO.md](DEMO_SCENARIO.md) の「Q&A 想定問答」を参照。
