このディレクトリには、IT資産監査・改善サービスのデモンストレーション用の資産が含まれています。

```
demo/
```

## 🎯 デモの目的

このデモパッケージは、以下の目的で使用できます：

1. **顧客へのサービス説明**: 実際のツールを動かして、具体的な価値を示す
2. **社内トレーニング**: 営業担当者やエンジニアがサービスを理解する
3. **技術検証**: ツール群の動作確認と改善点の洗い出し
4. **提案書作成**: デモ結果をもとに、具体的な提案書を作成

## 🎬 デモの実施方法

1. ツール群のアーキテクチャ説明

4. セキュリティとプライバシーの説明
5. CI/CD統合の可能性

**準備時間**: 1時間
**実施時間**: 60分
**期待される成果**: 技術的な承認と予算承認への推薦

### シナリオ3: 社内トレーニング

**想定**: 営業チームやエンジニアチームの研修

1. サービス概要の説明（30分）
2. デモの実施（20分）
3. 各ツールの詳細説明（30分）
4. Q&A（20分）

**準備時間**: 2時間
**実施時間**: 100分
**期待される成果**: 全員がデモを実施できるレベルに到達

## 💡 プレゼンのポイント

### 強調すべき3つの価値

1. **スピード**: AI活用で従来の半分の期間で完了
   - 例: 「この解析、手動なら2週間かかりますが、ツールなら10分です」

2. **品質**: 人間によるレビューで品質保証
   - 例: 「AIが検出した問題を、シニアエンジニアが全てレビューします」

3. **安心**: 瑕疵担保責任で不安を解消
   - 例: 「もし弊社の修正に問題があれば、3ヶ月間無償対応します」

### よくある質問と回答

**Q: AIが実装したコードは信頼できるのか？**
A: 4層の品質ゲート（自動検証→エンジニアレビュー→統合テスト→顧客検証）で品質を保証します。

**Q: 機密情報は安全か？**
A: NDA締結、アクセス制限、暗号化、監査ログで対応。プロジェクト終了後はデータを削除します。

**Q: 他社より料金が高いのでは？**
A: 診断と実装を別々に発注すると280万円。弊社は一気通貫で120万円なので、実は57%安いです。

詳細は [DEMO_SCENARIO.md](DEMO_SCENARIO.md) の「Q&A 想定問答」を参照。
