## セキュリティ問題（24件）

### Critical（7件）

#### 1. SECRET_KEYのハードコーディング
- **ファイル**: `ec_project/settings.py:9`
- **問題**: SECRET_KEYがソースコードに平文で記載
- **影響**: セッション改ざん、CSRF保護の無効化、暗号署名の偽造
- **修正工数**: 2時間
- **修正方法**: 環境変数に移行、django-environの使用

#### 2. DEBUG=True in production
- **ファイル**: `ec_project/settings.py:12`
- **問題**: DEBUG=Trueで本番運用
- **影響**: 詳細なエラー情報の漏洩、内部構造の露出
- **修正工数**: 1時間
- **修正方法**: 環境変数でDEBUG設定、本番はFalse

#### 3. CORS_ALLOW_ALL_ORIGINS = True
- **ファイル**: `ec_project/settings.py:47`
- **問題**: すべてのオリジンからのリクエストを許可
- **影響**: CSRF攻撃、データ窃取
- **修正工数**: 2時間
- **修正方法**: 許可するオリジンをホワイトリスト化

#### 4. SQLインジェクション脆弱性
- **ファイル**: `shop/views.py:13-23`
- **問題**: ユーザー入力を直接SQLに結合
- **影響**: データベース改ざん、情報漏洩、サーバー侵害
- **修正工数**: 3時間
- **修正方法**: ORMの使用、パラメータ化クエリ

#### 5. ユーザーIDの改ざん可能性
- **ファイル**: `shop/views.py:36-37`
- **問題**: GETパラメータからuser_idを取得
- **影響**: 他人のカートへの不正アクセス、なりすまし
- **修正工数**: 2時間
- **修正方法**: request.userの使用、認証チェック

#### 6. HTTPで決済API呼び出し + SSL検証無効化
- **ファイル**: `shop/views.py:98-107`
- **問題**: HTTPSではなくHTTP使用、verify=False
- **影響**: クレジットカード情報の盗聴、中間者攻撃
- **修正工数**: 2時間
- **修正方法**: HTTPS使用、SSL検証有効化

#### 7. パスワードのMD5ハッシュ化
- **ファイル**: `shop/views.py:113-126`
- **問題**: MD5は暗号学的に安全でない
- **影響**: レインボーテーブル攻撃、パスワード漏洩
- **修正工数**: 4時間
- **修正方法**: Djangoの組み込み認証（PBKDF2）使用
