## 概要

### 課題

調査フェーズにおけるステークホルダーヒアリングでは、多様な情報源から大量のデータを収集する必要があります：

```typescript
interface DiscoveryPhaseInputs {
  // 収集すべき情報
  stakeholderNeeds: {
    現状の課題: string[];
    改善したいポイント: string[];
    予算感: number;
    期待する効果: string[];
  };

  developmentContext: {
    開発体制: {
      担当者: string[];
      役割分担: Record<string, string>;
    };
    運用フロー: {
      デプロイ手順: string[];
      テスト体制: string;
    };
  };

  technicalDetails: {
    技術スタック: string[];
    システム構成: string;
    既存の問題: string[];
  };
}
```

### 目的

LLMを活用して、これらの情報を効率的に整理・構造化し、次フェーズ（分析・診断）で利用可能な形式に変換する。

### 設計方針

1. **マルチベンダー戦略**: 各LLMの得意分野を活かす
2. **Google Workspace統合**: 既存ワークフローとの親和性
3. **段階的処理**: 生データ → 中間データ → 構造化データ
4. **トレーサビリティ**: すべての処理過程を記録（監査証跡）

## 入力データ種別

### データソースマトリクス

| データソース | 形式 | 頻度 | 情報密度 | 処理優先度 |
|------------|------|------|---------|-----------|
| **ドキュメント** | PDF, Word, Excel, PPT, 画像 | 初回一括 | 高 | 1 |
| **Web会議** | MP4 録画 + 文字起こし | 週1-2回 | 中 | 2 |
| **メール** | Gmail (HTML/Text) | 日次 | 低-中 | 3 |
| **Slack/Teams** | メッセージログ | 常時 | 低 | 4 |
| **既存システム** | コード、DB、設定ファイル | 初回一括 | 高 | 1 |

## データベース設計

### テーブル設計

```sql
-- プロジェクト定義情報（統合後）
CREATE TABLE projects (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID NOT NULL REFERENCES customers(id),
    name VARCHAR(255) NOT NULL,
    status VARCHAR(50) NOT NULL, -- 'discovery', 'analysis', 'diagnosis', etc.

    -- ディスカバリーフェーズの成果物
    discovery_summary JSONB, -- 統合レポート（JSON全体）

    -- メタデータ
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    completed_phases TEXT[], -- ['discovery', 'analysis']

    -- データ品質指標
    data_completeness FLOAT, -- 0.0-1.0
    data_confidence FLOAT    -- 0.0-1.0
);

-- ヒアリング情報（生データ + 処理済みデータ）
CREATE TABLE hearings (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id),

    -- データソース種別
    source_type VARCHAR(50) NOT NULL, -- 'document' | 'meeting' | 'email' | 'repository'

    -- 元データ
    original_file_url TEXT, -- GCS or Google Drive URL
    original_file_name VARCHAR(255),
    mime_type VARCHAR(100),

    -- 処理結果
    extracted_text TEXT, -- OCR or 文字起こしの生テキスト
    structured_data JSONB, -- LLMで抽出した構造化データ

    -- メタデータ
    processed_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    processed_by VARCHAR(100), -- 'google-document-ai' | 'claude-3.5-sonnet' | etc
    processing_cost DECIMAL(10, 4), -- APIコスト（ドル）

    -- 品質指標
    confidence FLOAT, -- 0.0-1.0
    needs_human_review BOOLEAN DEFAULT FALSE,
    review_reason TEXT,
    reviewed_by VARCHAR(100),
    reviewed_at TIMESTAMP WITH TIME ZONE,

    -- 検索用
    search_vector tsvector, -- 全文検索用

    CONSTRAINT hearings_project_fk FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

-- 全文検索インデックス
CREATE INDEX hearings_search_idx ON hearings USING GIN(search_vector);

-- メールスレッド（特別扱い）
CREATE TABLE email_threads (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id),

    thread_id VARCHAR(255) NOT NULL, -- Gmail thread ID
    subject TEXT NOT NULL,
    participants JSONB, -- [{"email": "...", "name": "..."}]

    messages JSONB, -- メッセージの配列
    qa_pairs JSONB, -- 抽出されたQ&A

    first_message_at TIMESTAMP WITH TIME ZONE,
    last_message_at TIMESTAMP WITH TIME ZONE,

    processed_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- ディスカバリーフェーズのアクションアイテム
CREATE TABLE discovery_action_items (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id),

    description TEXT NOT NULL,
    category VARCHAR(50), -- 'follow_up_question' | 'missing_document' | 'clarification_needed'
    priority INTEGER, -- 1-5

    source_hearing_id UUID REFERENCES hearings(id), -- どのヒアリングから生まれたか

    status VARCHAR(50) DEFAULT 'open', -- 'open' | 'in_progress' | 'resolved'
    assigned_to VARCHAR(100),

    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    resolved_at TIMESTAMP WITH TIME ZONE
);

-- 監査ログ（すべての処理過程を記録）
CREATE TABLE audit_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID REFERENCES projects(id),

    event_type VARCHAR(100) NOT NULL, -- 'hearing_ingested' | 'llm_processed' | 'human_reviewed' | etc
    event_data JSONB,

    user_id VARCHAR(100), -- 人間の操作の場合
    system_component VARCHAR(100), -- 'document-pipeline' | 'meeting-pipeline' | etc

    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- インデックス
CREATE INDEX hearings_project_id_idx ON hearings(project_id);
CREATE INDEX hearings_source_type_idx ON hearings(source_type);
CREATE INDEX email_threads_project_id_idx ON email_threads(project_id);
CREATE INDEX discovery_action_items_project_id_idx ON discovery_action_items(project_id);
CREATE INDEX audit_logs_project_id_idx ON audit_logs(project_id);
```
