# 再利用可能ツール群 - 要件ヒアリングメモ

## 共通要件

### 全ツール共通で必要なこと
- プロジェクトごとにデータを管理できること
- 複数のプロジェクトを並行して扱えること
- データのエクスポート・インポートができること（別環境への移行用）
- 日本語・英語両方に対応すること（将来の海外展開を見据えて）
- オフライン作業可能（顧客環境に直接持ち込む場合がある）
- ログをしっかり残すこと（瑕疵担保対応時のエビデンス）

### データ保存方針
- SQLiteかPostgreSQLか？ → まずはSQLiteで小さく始める？
- ファイルベースでも良い部分は？（JSON、Markdown等）
- バックアップは自動？手動？

### セキュリティ
- 顧客のソースコードを扱うのでセキュアに
- アクセス制限は必要？（複数人で使う場合）
- 顧客情報の取り扱い（GDPR、個人情報保護法対応）

## Tool 11: 見積もり計算機

### やりたいこと
- Issueから工数を自動見積もりしたい
- 見積もりの根拠を示したい
- 過去実績から精度を上げたい

### 見積もり要素
- 基本工数（Issue重要度ベース）
- 複雑度係数（コード変更範囲、依存関係）
- リスク係数（不確実性）
- バッファ（20-30%）

### 計算方法
```
見積工数 = 基本工数 × 複雑度係数 × リスク係数 × (1 + バッファ率)
```

### 過去実績の活用
- 実績工数を記録
- 見積もりと実績の差分を分析
- 係数を自動調整（機械学習？）

### 出力
- 最小工数、最大工数、期待工数
- 信頼区間
- 見積もりの根拠（何が工数を押し上げているか）

### 疑問点

## Tool 12: サンドボックス環境構築ツール

### やりたいこと
- 解析対象のシステムを安全に動作させる環境を自動構築したい
- 本番環境に影響を与えずに改善案の動作確認・テストをしたい
- 言語・フレームワークに依存しない汎用的な環境構築がしたい
- 既存のDocker/K8s環境があればそのまま活用したい
- 古いオンプレシステムでもコンテナ化して動かしたい

### 対応パターン

**パターン1: 既存のコンテナ環境を活用**
- docker-compose.ymlが存在する場合 → そのまま利用
- Dockerfileがある場合 → ビルドして起動
- Kubernetes (K8s) マニフェストがある場合 → ローカルK8s (minikube/kind) で起動
- Helmチャートがある場合 → Helm経由でデプロイ

**パターン2: 自動検出してコンテナ化**
```typescript
interface AutoDetectionRules {
  Node_js: {
    検出条件: "package.json の存在",
    ベースイメージ: "node:18-alpine",
    起動コマンド: "npm install && npm start",
    公開ポート: 3000
  },
  PHP: {
    検出条件: "composer.json または *.php の存在",
    ベースイメージ: "php:8.2-apache または php:8.2-fpm + nginx",
    起動コマンド: "composer install && php -S 0.0.0.0:8000",
    公開ポート: 8000
  },
  Python: {
    検出条件: "requirements.txt または Pipfile",
    ベースイメージ: "python:3.11-slim",
    起動コマンド: "pip install -r requirements.txt && python app.py",
    公開ポート: 5000
  },
  DotNet: {
    検出条件: "*.csproj または *.sln",
    ベースイメージ: "mcr.microsoft.com/dotnet/aspnet:8.0",
    起動コマンド: "dotnet restore && dotnet run",
    公開ポート: 5000
  },
  Java: {
    検出条件: "pom.xml (Maven) または build.gradle (Gradle)",
    ベースイメージ: "eclipse-temurin:21-jdk",
    起動コマンド: "mvn clean install && java -jar target/*.jar",
    公開ポート: 8080
  },
  Ruby: {
    検出条件: "Gemfile",
    ベースイメージ: "ruby:3.2-alpine",
    起動コマンド: "bundle install && rails server",
    公開ポート: 3000
  },
  Static: {
    検出条件: "index.html のみで動的言語なし",
    ベースイメージ: "nginx:alpine",
    起動コマンド: "nginx -g 'daemon off;'",
    公開ポート: 80
  }
}
```

**パターン3: オールインワンLinuxコンテナ（レガシー対応）**
- オンプレサーバで動いている古いシステム向け
- Apache + PHP + MySQL を一つのコンテナに同梱
- systemdが必要な場合はsystemd対応イメージを使用
- 手動セットアップ手順書をもとにDockerfileを自動生成

### 主要機能

**環境自動検出**
```bash
# プロジェクトを解析して最適な環境を提案
sandbox detect --path ./target-app

# 出力例:
# Detected: Node.js 16.x (package.json found)
# Framework: Express.js
# Database: PostgreSQL (detected in docker-compose.yml)
# Recommended setup: docker-compose
```

**環境構築**
```bash
# 自動検出して環境を構築
sandbox build --path ./target-app --output ./sandbox-env

# 特定のパターンを指定
sandbox build --path ./target-app --type php-apache --output ./sandbox-env

# 既存のDocker環境を使用
sandbox build --path ./target-app --use-existing-docker
```

**環境起動・停止**
```bash
# 環境を起動
sandbox up --env ./sandbox-env

# バックグラウンドで起動
sandbox up --env ./sandbox-env --detach

# 環境を停止
sandbox down --env ./sandbox-env

# 環境をリセット（データを含めて削除）
sandbox reset --env ./sandbox-env
```

**動作確認・ヘルスチェック**
```bash
# 環境が正常に起動しているか確認
sandbox health --env ./sandbox-env

# ログの確認
sandbox logs --env ./sandbox-env --follow

# コンテナ内でコマンド実行
sandbox exec --env ./sandbox-env --command "php artisan migrate"
```

**スナップショット管理**
```bash
# 現在の状態を保存
sandbox snapshot create --env ./sandbox-env --name "before-change"

# スナップショットから復元
sandbox snapshot restore --env ./sandbox-env --name "before-change"

# スナップショット一覧
sandbox snapshot list --env ./sandbox-env
```

### データベース対応

**パターン1: 既存のDBコンテナを利用**
- docker-compose.ymlにDB定義がある → そのまま使用
- 環境変数から接続情報を取得

**パターン2: 専用DBコンテナを自動追加**
```yaml
# 自動生成されるdocker-compose.yml例
services:
  app:
    build: .
    ports:
      - "8000:8000"
    environment:
      - DB_HOST=db
      - DB_PORT=5432
    depends_on:
      - db

  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_DB=testdb
      - POSTGRES_USER=testuser
      - POSTGRES_PASSWORD=testpass
    volumes:
      - db-data:/var/lib/postgresql/data
      - ./db-init:/docker-entrypoint-initdb.d  # 初期データ投入

volumes:
  db-data:
```

**パターン3: 本番DBのダンプを使用**
```bash
# 本番DBのダンプファイルからサンドボックスDBを構築
sandbox build --path ./target-app --db-dump ./production-dump.sql
```
