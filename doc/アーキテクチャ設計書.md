## 概要

### ビジネス要件
- 初年度目標: 12件の顧客（月1件ペース）
- サービス期間: 1顧客あたり3-5ヶ月
- 並行対応: 同時3-5プロジェクト
- チーム規模: 2-5名（エンジニア）
- 顧客共有: メトリクス・レポートのリアルタイム共有が必要

### 技術的要件
- **環境独立性**: 開発者のローカル環境に依存しない
- **再現性**: 同じ環境を何度でも構築できる
- **スケーラビリティ**: 顧客数増加に対応できる
- **セキュリティ**: 顧客のソースコードを安全に扱う
- **監査証跡**: すべての操作をログに記録

## 現状の課題

### 1. 環境依存問題

**better-sqlite3のビルドエラー**
```bash
# Windows環境でのエラー例
Error: Could not find any Python installation to use
gyp ERR! stack Error: `gyp` failed with exit code: 1
```

**影響範囲**:
- `@it-supervisor/metrics-model` (メトリクスDB)
- `@it-supervisor/issue-manager` (Issue管理)
- 上記に依存する全ツール

**根本原因**:
- ネイティブモジュールのビルドにPython/node-gypが必要
- 開発者ごとに環境が異なる

### 2. チーム共有の課題

**現状の問題**:
- 各自のローカルでSQLiteファイルを管理
- Gitでバイナリファイル（.db）を共有するのは非効率
- 同時編集時の競合リスク

**必要な機能**:
- 複数人で同じプロジェクトデータを参照
- リアルタイムでの進捗共有
- ロールベースのアクセス制御

### 3. 顧客共有の課題

**現状の問題**:
- レポートをPDF/HTMLでエクスポートしてメール送付
- メトリクスの推移をリアルタイムで見せられない
- 顧客からのフィードバック収集が困難

**必要な機能**:
- Webベースのダッシュボード
- 顧客専用のログイン機能
- Before/Afterのグラフ表示
- レポートのオンライン共有

## 技術選定方針

### docker-compose vs k8s vs ECS

| 観点 | docker-compose | ECS/Fargate | Kubernetes |
|------|---------------|-------------|-----------|
| **学習コスト** | 低 | 中 | 高 |
| **運用コスト** | 低 | 中 | 高 |
| **スケーラビリティ** | 低 | 中 | 高 |
| **適用規模** | 開発環境 | 中小規模本番 | 大規模本番 |
| **マネージド度** | なし | 高 | 中（EKS使用時）|
| **初期コスト** | $0 | 低 | 中-高 |

### 段階的移行戦略


### docker-compose.yml 設計

```yaml
version: '3.9'

services:
  # メインアプリケーション
  app:
    build:
      context: .
      dockerfile: docker/app/Dockerfile
    container_name: it-supervisor-app
    volumes:
      - ./tools:/app/tools:ro           # ツール群（読み取り専用）
      - ./data:/app/data                # データ永続化
      - ./reports:/app/reports          # レポート出力
      - customer-repos:/app/repos       # 顧客リポジトリ（分析対象）
    environment:
      - NODE_ENV=development
      - DATABASE_URL=postgresql://postgres:postgres@db:5432/it_supervisor
      - REDIS_URL=redis://redis:6379
    depends_on:
      - db
      - redis
    networks:
      - it-supervisor-network

  # PostgreSQL（SQLiteからの移行）
  db:
    image: postgres:16-alpine
    container_name: it-supervisor-db
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: it_supervisor
    volumes:
      - postgres-data:/var/lib/postgresql/data
      - ./docker/db/init.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "5432:5432"
    networks:
      - it-supervisor-network

  # Redis（分析ジョブのキュー管理）
  redis:
    image: redis:7-alpine
    container_name: it-supervisor-redis
    ports:
      - "6379:6379"
    networks:
      - it-supervisor-network

  # ESLint分析用コンテナ
  analyzer-eslint:
    build:
      context: .
      dockerfile: docker/analyzers/eslint.Dockerfile
    container_name: it-supervisor-analyzer-eslint

      - customer-repos:/repos:ro
    networks:
      - it-supervisor-network

      - SONAR_JDBC_URL=jdbc:postgresql://db:5432/sonarqube
```

### Dockerfile設計

#### メインアプリケーション (docker/app/Dockerfile)

```dockerfile
FROM node:20-alpine AS builder

# ビルドツールのインストール（better-sqlite3ビルド用）
RUN apk add --no-cache python3 make g++

WORKDIR /app

# 依存関係のインストール
COPY tools/package*.json ./
RUN npm ci

# ソースコードのコピー
COPY tools/ ./

# TypeScriptビルド
RUN npm run build

# プロダクション用の軽量イメージ
FROM node:20-alpine

WORKDIR /app

# 本番依存関係のみインストール
COPY tools/package*.json ./
RUN npm ci --omit=dev

# ビルド成果物のコピー
COPY --from=builder /app/packages/ ./packages/

# ヘルスチェック
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:3000/health', (r) => {process.exit(r.statusCode === 200 ? 0 : 1)})"

# エントリーポイント
CMD ["node", "packages/cli/dist/index.js"]
```

## データアーキテクチャ

### SQLite → PostgreSQLへの移行

**理由**:
- 複数人での同時アクセス
- トランザクション性能
- AWS RDSとの親和性
- バックアップ・復旧の容易さ

### データモデル

```sql
-- プロジェクトテーブル
CREATE TABLE projects (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID NOT NULL REFERENCES customers(id),
    name VARCHAR(255) NOT NULL,
    repository_url TEXT,
    status VARCHAR(50) NOT NULL, -- 'discovery', 'analysis', 'diagnosis', etc.
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- メトリクステーブル
CREATE TABLE metrics (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id),
    category VARCHAR(100) NOT NULL, -- 'code_quality', 'security', 'performance'
    name VARCHAR(255) NOT NULL,
    value JSONB NOT NULL,
    measured_at TIMESTAMP WITH TIME ZONE NOT NULL,
    phase VARCHAR(50), -- 'before', 'after'
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Issueテーブル
CREATE TABLE issues (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id),
    title TEXT NOT NULL,
    description TEXT,
```

### データ移行スクリプト

```typescript
// tools/scripts/migrate-sqlite-to-postgres.ts
import Database from 'better-sqlite3';
import { Pool } from 'pg';

async function migrate() {
  const sqlite = new Database('./data/metrics.db');
  const pg = new Pool({
    connectionString: process.env.DATABASE_URL
  });

  try {
    // プロジェクトの移行
    const projects = sqlite.prepare('SELECT * FROM projects').all();
    for (const project of projects) {
      await pg.query(
        'INSERT INTO projects (id, name, status, created_at) VALUES ($1, $2, $3, $4)',
        [project.id, project.name, project.status, project.created_at]
      );
    }

    // メトリクスの移行
    const metrics = sqlite.prepare('SELECT * FROM metrics').all();
    for (const metric of metrics) {
      await pg.query(
        'INSERT INTO metrics (id, project_id, category, name, value, measured_at) VALUES ($1, $2, $3, $4, $5, $6)',
        [metric.id, metric.project_id, metric.category, metric.name, JSON.parse(metric.value), metric.measured_at]
      );
    }

    console.log('Migration completed successfully');
  } catch (error) {
    console.error('Migration failed:', error);
    throw error;
  } finally {
    sqlite.close();
    await pg.end();
  }
}

migrate();
```
