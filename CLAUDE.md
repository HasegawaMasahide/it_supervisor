# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## プロジェクト概要

このリポジトリは「IT資産監査・改善サービス」の事業計画と関連資産を管理しています。中小企業の既存IT資産（業務アプリ、コーポレートサイト、CMS等）に対し、Agent AI（Claude Code等）を活用した高速監査・改善提案・実装をワンストップで提供するサービスです。

**主要な差別化要素:**
- AI+人間のハイブリッド体制による効率化と品質保証
- 瑕疵担保責任による顧客不安の解消
- 診断から実装・効果測定までの一気通貫対応

## アーキテクチャと設計思想

### サービスアーキテクチャ

このサービスは7つのフェーズで構成されています:

1. **調査 (Discovery)**: システム構成・技術スタック把握 (1-2週間)
2. **分析 (Analysis)**: AI活用した静的解析・メトリクス収集 (1週間)
3. **診断 (Diagnosis)**: 人間による課題の優先順位付けとリスク評価 (1週間)
4. **提案 (Proposal)**: 改善案作成・ROI試算 (1週間)
5. **実装 (Implementation)**: コード改善・テスト・デプロイ (4-8週間)
6. **効果測定 (Measurement)**: Before/After比較 (1-2週間)
7. **報告 (Reporting)**: 最終レポート作成 (3日)

### ツール群のデータフロー

[doc/ツール群の要件ヒアリングメモ.md](doc/ツール群の要件ヒアリングメモ.md)に詳述されている11種のツールは以下のフローで連携:

```
リポジトリ解析 → 静的解析 → メトリクスDB
                              ↓
Issue管理 ← 人間の診断 ← 分析結果の可視化
    ↓
改善提案生成 → 見積もり計算
    ↓
変更管理 → 実装 → 効果測定 → メトリクスDB
    ↓
レポート生成 ← チャート生成 ← メトリクスDB
    ↓
ダッシュボード
```

## ランディングページの開発

### 開発環境

静的HTMLサイトのため特別なビルドプロセスは不要:

```bash
# ローカルで確認（任意のHTTPサーバーを使用）
# 例: Python HTTP Server
cd lp
python -m http.server 8000

# または: Node.js http-server
npx http-server lp -p 8000
```

ブラウザで http://localhost:8000 にアクセス

### デザイン原則

- **日本語対応**: Noto Sans JP フォント使用
- **レスポンシブ**: モバイルファーストデザイン（768px以下でモバイルメニュー表示）
- **アクセシビリティ**: セマンティックHTML、適切なaria-label
- **パフォーマンス**: 画像遅延読み込み、スムーススクロール

### 主要コンポーネント

- **ヒーローセクション**: 3つの統計数値（AI活用率60%、瑕疵担保期間3ヶ月、平均コスト削減200万円）
- **課題セクション**: 顧客ペインポイント4項目（工数不足、説明責任、属人化リスク、AI活用不安）
- **特徴セクション**: 3つの強み（ハイブリッド体制、瑕疵担保、ワンストップ）
- **ツールセクション**: 12種の専用ツールと活用技術、4層品質ゲート
- **プロセスセクション**: 6段階のサービスフロー（クリックで詳細展開）
- **改善事例セクション**: 4つのデモアプリ事例（Laravel/React/Django/ASP.NET）- クリックで詳細モーダル表示
- **料金プランセクション**: 3プラン比較表
- **お客様の声セクション**: 3つの導入事例（クリックで詳細モーダル表示）
- **FAQアコーディオン**: よくある質問6項目
- **お問い合わせフォーム**: 現在はダミー実装（console.log出力）

### コード編集時の注意点

**[lp/script.js](lp/script.js):**
- フォーム送信は現在ダミー実装（149-188行目）- 実際のAPI連携が必要な場合はコメント部分を参考に実装
- スクロールアニメーションは Intersection Observer API を使用（193-269行目）
- カウンターアニメーションは統計数値に適用（227-269行目）

**[lp/styles.css](lp/styles.css):**
- CSSカスタムプロパティ未使用 - 色やフォントサイズが直接指定されているため、テーマ変更時は複数箇所の修正が必要
- モバイル対応の一部は[lp/script.js](lp/script.js)内で動的に追加（75-123行目）

- Claude API活用（プロンプト管理とコスト管理が課題）
